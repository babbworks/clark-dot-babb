Software layer

Clark builds tools that serve the work, not the other way around.

The software layer supports Learning and Working. It does not replace them. Clark's tools exist to record what happens on the floor, make that record trustworthy, and connect facilities into a coordinated network. The commitment is to open source industrial software that any centre, node, or regional actor can adopt.

Open source industrial tools

Clarkware

Clarkware is the name for every piece of software Clark builds. It is a family of tools — not a monolithic application — designed for real industrial environments. The tools are open source, extension-based, and built so that any facility can run them independently.

Clarkware covers production workstations, training delivery, network coordination, and facility reporting. Each tool has a clear scope and can be adopted on its own.

In production development

Clark IPE

The Integrated Production Environment is the first and primary Clarkware product. It is a Theia-based workstation application that unifies job context, IPC inspection, firmware workflows, evidence capture, and network coordination at the bench.

The IPE is already in active development. The current build boots the Theia shell, renders four primary panels, and supports profile management. It is being developed against Theia 1.69.

Other tools that support operations

Clark Courses platform

Training delivery and certification tracking for IPC-aligned industrial programmes. Designed to work alongside the IPE at centres, but deployable independently by any training provider.

Network coordination

Cross-node job routing, corridor-level visibility, and audit aggregation. Built on IPC-CFX messaging so that any standards-compliant system can participate without custom integration work.

Facility reporting

Shift reports, quality summaries, and production dashboards derived from workstation-level event streams. Built from evidence, not from separate data entry.

What the tools are not

Not an ERP replacement. Clarkware is a workstation-first evidence system, not a business management platform.

Not a generic chat app with a file tree. The IPE is purpose-built for production, inspection, and firmware work.

Not a sprawling platform before one centre works. The tools prove themselves at a single facility before scaling to the network.

Why it matters

The software makes the public record trustworthy.

Clark Tools are the mechanism that turns floor activity into structured, auditable, network-legible evidence. That evidence is what makes Clark's public claims inspectable rather than decorative.
