# Roadmap

This is the first-pass sequence for turning the IPE idea into a credible open source workbench.

## Phase 0: Align the frame

Goals:

- confirm the IPE definition
- confirm the first wedge
- inventory what already exists in `v2ww`
- identify the minimum pilot workflow

Exit criteria:

- a short product definition exists
- the first user journey is written down
- the major unknowns are listed

## Phase 1: Bench shell and context

Goals:

- establish the IPE workspace shell
- create the active job view
- create notes and evidence capture primitives
- create station and equipment context surfaces

Exit criteria:

- a user can open a job and see the correct context
- notes can be attached to a job and station
- artifacts can be linked to the record

## Phase 2: Electronics workflow MVP

Goals:

- support at least one assembly, flash, or repair workflow end to end
- ingest one meaningful test or device artifact type
- support a review or handoff step
- provide exportable history

Exit criteria:

- one real workflow runs through the system without manual reconstruction
- evidence is attached to the workflow record
- the resulting record can be reviewed by a second person

## Phase 3: Bench integrations

Goals:

- add more device and test integrations
- improve procedure handling
- support richer equipment metadata
- refine the audit trail and search

Exit criteria:

- the system is useful across multiple benches or stations
- common bench tools can be integrated without custom one-off hacks

## Phase 4: Collaboration and scale

Goals:

- remote review and expert support
- permissioning and role boundaries
- reporting and export pipelines
- packaging and deployment improvements

Exit criteria:

- the IPE works as a shared industrial system, not just a local utility

## Phase 5: Platform maturity

Goals:

- open plugin model
- reusable industrial primitives
- docs and example deployments
- community contribution path

Exit criteria:

- third parties can extend the workbench without deep core changes
