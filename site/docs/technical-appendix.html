System boundary

The technical layer supports Learning and Working; it does not replace them.

Clarkware is the software family behind the evidence layer. The IPE is the workstation-level environment where work, context, and record meet.

The shell

A thin Theia-based workstation shell gives the user workspace navigation, dockable panels, command access, and extension support.

The services

Jobs, notes, evidence capture, station metadata, test results, messaging, and reporting belong in services and backend modules.

The adapters

Serial tools, watched folders, firmware flashing, test export ingestion, and local databases connect the bench to the system.

The record

Append-first event history, job-linked artifacts, and clear human/AI/automation distinctions preserve the audit trail.

What is not in v1

Not an ERP replacement.

Not a generic chat app with a file tree.

Not a sprawling platform before one centre works.

Why it matters

The software makes the public record trustworthy.

That is why it belongs in the appendix and not on the front page. The public argument stays simple; the technical basis remains available.
